# Support

## Где задавать вопросы

- Для багов и предложений: GitHub Issues.
- Для обсуждения архитектуры/подхода: GitHub Discussions (если включены) или issue с меткой `question`.
- Для security-вопросов: процесс из `SECURITY.md`.

## Что прикладывать к обращению

Чтобы ускорить разбор, добавьте:

- версию/commit или ветку;
- среду запуска (OS, Python, Node, Docker);
- шаги воспроизведения;
- ожидаемое и фактическое поведение;
- логи и traceback (без секретов и персональных данных);
- скриншоты/видео для UI-проблем.

## SLA (Best Effort)

Поддержка осуществляется в режиме best effort, без формальных SLA.
Критичные security-вопросы обрабатываются в приоритетном порядке.
